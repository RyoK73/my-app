# コンポーネントのファイル分割方針

## 背景・問題意識

コンポーネントを作るとき、どの単位でファイルにまとめるべきか基準がなかった。
特に `PostList` と `TagList` のように「似ている」コンポーネントを同じファイルにまとめるべきか迷った。

## 出た案・選択肢

- 似たコンポーネントを1ファイルにまとめる（例: list系をまとめる）
- 1コンポーネント1ファイルに分ける

## 反論・懸念点

- `PostList` と `TagList` は構造が似ているだけで、変更理由が独立している
- 同じファイルにまとめると、片方だけ触りたいときにもう片方も目に入り認知負荷が上がる

## 決定・方向性

**1コンポーネント1ファイル**を基本方針とする。

判断基準：**「一方を変えるとき、もう一方も変える可能性が高いか？」**
- Yes → まとめてもよい
- No → 別ファイルにする

例外として、**親と密結合した小さなサブコンポーネント**（例: `PostList` の中でしか使わない `PostListItem`）は同じファイルに入れてよい。
